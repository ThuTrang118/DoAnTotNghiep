==================================================================================
OUTPUT FORMAT - STRICT JSON ONLY
==================================================================================

NHIỆM VỤ: Chỉ trả về duy nhất 1 JSON object. Không giải thích, không markdown.

RÀNG BUỘC VỀ DỮ LIỆU TRONG JSON:
1. Inputs: Phải điền giá trị cụ thể (Literal), KHÔNG dùng từ mô tả "invalid", "valid".
2. Technique: Chỉ dùng "EP", "BVA", "DT".
3. BoundaryMeta: 
   - Phải đầy đủ cho mọi item BVA.
   - "reference_value": Giá trị số của ngưỡng biên đang xét.
4. Objective: Phải nêu rõ lý do lỗi (Ví dụ: "Kiểm tra Password độ dài tối thiểu - 1").

JSON SCHEMA:
{
  "feature": "[Tên chức năng]",
  "coverage_summary": {
    "EP_count": 0,
    "BVA_count": 0,
    "DT_count": 0,
    "TOTAL": 0
  },
  "items": [
    {
      "Testcase": "[ID]",
      "Technique": "EP|BVA|DT",
      "Objective": "Mục tiêu đơn tử",
      "BoundaryMeta": {
        "field": "tên field",
        "rule_type": "range|fixed_length|max_only|min_only",
        "boundary_point": "min-1|min|min+1|max-1|max|max+1|N-1|N|N+1",
        "reference_value": 0
      },
      "Inputs": { "field1": "value", "field2": "value" },
      "Expected": "Thông báo lỗi chính xác từ đặc tả"
    }
  ]
}

BẢNG KIỂM TRA TRƯỚC KHI XUẤT:
- Range [8,20] có đủ 6 case BVA chưa?
- Fixed-length 10 có đủ 3 case BVA (9, 10, 11) chưa?
- Các field Required đã có case EMPTY riêng lẻ chưa?
- ConfirmPassword đã có case "không trùng" (DT) chưa?
- Case Valid (Thành công) đã có chưa?
