==================================================================================
CẤM TUYỆT ĐỐI
==================================================================================

Không được output:
- markdown
- code fence
- comment kiểu // hoặc /* */
- giải thích
- ghi chú
- text trước JSON
- text sau JSON
- key ngoài schema quy định

==================================================================================
JSON SCHEMA DUY NHẤT ĐƯỢC PHÉP
==================================================================================

{
  "feature": "",
  "items": [
    {
      "Testcase": "",
      "Technique": "DT|EP|BVA",
      "Objective": "",
      "BoundaryMeta": null,
      "Inputs": {
        {{ITEM_FIELDS_SCHEMA}}
      },
      "Expected": ""
    }
  ]
}

==================================================================================
QUY TẮC CHUNG
==================================================================================

1. Mỗi item = đúng 1 coverage item
2. Mỗi item negative = đúng 1 lỗi chính
3. Nếu đặc tả có success -> phải có ít nhất 1 testcase success
4. Chỉ sinh testcase khi expected xác định được từ đặc tả

==================================================================================
INPUTS
==================================================================================

- Phải đủ field
- Không thêm field
- Không thiếu field
- Dữ liệu phải là giá trị thật

CẤM:
- "valid", "invalid"
- "min", "max"
- "N-1", "N+1"

==================================================================================
EXPECTED
==================================================================================

- Phải đúng EXACT theo đặc tả
- Không tự sửa câu
- Không tự bịa

==================================================================================
TECHNIQUE RULE
==================================================================================

DT:
- required
- relation
- logic nhiều field

EP:
- format
- prefix
- pattern
- equivalence class

BVA:
- boundary
- min/max
- fixed length

CẤM:
- DT cho data class
- EP cho boundary
- BVA khi không có biên

==================================================================================
QUY TẮC PHÂN TÁCH
==================================================================================

1. Same expected ≠ same testcase  
→ phải tách riêng từng invalid class

2. Có biên → bắt buộc dùng BVA

3. Không được:
- gộp nhiều lỗi
- gộp nhiều class
- tạo testcase chung kiểu "không hợp lệ"

==================================================================================
BVA RULE
==================================================================================

Chỉ dùng khi đặc tả có:
- từ X đến Y
- đúng N
- tối đa / tối thiểu

BoundaryMeta phải đúng:

{
  "field": "",
  "rule_type": "",
  "boundary_point": "",
  "reference_value": 0
}

BoundaryMeta chỉ được dùng khi testcase thực sự là boundary coverage.

BẮT BUỘC:
- reference_value = mốc gốc
- boundary_point = 1 giá trị
- input phải khớp boundary
- Nếu testcase thuộc BVA → BoundaryMeta KHÔNG được null
- Nếu testcase KHÔNG phải BVA → BoundaryMeta PHẢI là null

MAPPING BẮT BUỘC:

- boundary_point phải khớp với input thực tế
- reference_value phải đúng mốc gốc trong đặc tả

==================================================================================
ISOLATION
==================================================================================

- 1 testcase = 1 lỗi
- field khác phải valid

==================================================================================
VALID DATA
==================================================================================

- dùng dữ liệu đơn giản nhất
- không cần bao phủ hết pattern

==================================================================================
SELF-CHECK (KHÔNG IN RA)
==================================================================================

1. Có đủ required chưa
2. Có đủ invalid class chưa
3. Có relation chưa
4. Có success chưa
5. Có BVA nếu có biên chưa
6. Có tách invalid class chưa
7. Có sai schema không
