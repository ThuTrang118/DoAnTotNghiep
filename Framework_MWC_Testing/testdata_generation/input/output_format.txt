==================================================
YÊU CẦU ĐẦU RA (OUTPUT RULES)
==================================================
- Chỉ trả về duy nhất 1 JSON object hợp lệ.
- Không giải thích, không markdown, không văn bản thừa.
- JSON phải bắt đầu bằng { và kết thúc bằng }.

==================================================
JSON SCHEMA
==================================================
{
  "feature": "[Tên chức năng]",
  "coverage_summary": {
    "DT_count": 0,
    "EP_count": 0,
    "BVA_count": 0,
    "TOTAL": 0
  },
  "items": [
    {
      "Testcase": "[PREFIX]01",
      "Technique": "DT | EP | BVA",
      "Objective": "Mô tả rõ mục tiêu kiểm thử",
      "BoundaryMeta": null,
      "Inputs": {
{{ITEM_FIELDS_SCHEMA}}
      },
      "Expected": "Kết quả mong đợi"
    }
  ]
}

==================================================
RÀNG BUỘC GIÁ TRỊ FIELD

1. Field "Technique":
- Chỉ được nhận DUY NHẤT 1 trong 3 giá trị sau:
  - "DT"
  - "EP"
  - "BVA"
- Không được ghi:
  - "DT | EP | BVA"
  - "DT, EP, BVA"
  - "Decision Table"
  - "Equivalence Partitioning"
  - "Boundary Value Analysis"

2. Quy tắc gán "Technique":
- Nếu testcase được sinh từ logic tổ hợp điều kiện / trường bắt buộc / phụ thuộc giữa các field -> "DT"
- Nếu testcase đại diện cho 1 lớp dữ liệu hợp lệ hoặc không hợp lệ -> "EP"
- Nếu testcase kiểm tra giá trị biên min/max hoặc N-1/N/N+1 -> "BVA"

==================================================
KIỂM TRA TRƯỚC KHI XUẤT (SELF-CHECK)
1. Tính độc lập: Có đúng 1 trường lỗi cho mỗi ca kiểm thử Invalid không?
2. Tính chính xác: Các giá trị biên (ví dụ: 10 ký tự) có đúng là chuỗi 10 ký tự không?
3. Tính khớp: Kết quả Expected có khớp từng chữ với tài liệu nghiệp vụ không?
