Generate test cases for the LOGIN feature of the MWC website.

==================================================
FEATURE DESCRIPTION
==================================================

Login form has two fields:
- Username (required)
- Password (required)

Seed account for successful login:
- Username: "Ánh Dương Phạm"
- Password: "anhduong@123"

==================================================
SYSTEM BEHAVIOR
==================================================

1. If Username or Password is missing
→ show: "Vui lòng điền vào trường này"

2. If both fields are present but credentials are incorrect
→ show: "Tên đăng nhập hoặc mật khẩu không đúng!"

3. If Username and Password exactly match the seed account
→ login succeeds and Expected is:
"Ánh Dương Phạm"

==================================================
TEST DESIGN INSTRUCTION
==================================================

Use black-box testing techniques described in the knowledge base:
- Equivalence Partitioning
- Boundary Value thinking
- Cause-Effect reasoning
- Error Guessing

First identify the distinct behavioral groups of the Login feature.

Then generate a compact set of representative test cases that collectively cover:
- successful login
- missing required fields
- incorrect authentication
- representative equivalence classes of Username and Password
- boundary-like inputs (for example: empty, very short, unusually long)
- realistic error-prone inputs (for example: spaces, case differences, special characters)

Do not generate multiple test cases from the same equivalence class if they produce the same behavior and the same expected result, unless a new case adds a meaningful boundary or a new error-prone behavior.

Each new test case must add new behavioral coverage.

Stop generating when additional cases no longer add meaningful new coverage.

==================================================
EXPECTED RESULTS
==================================================

Expected must be one of:

"Ánh Dương Phạm"
"Vui lòng điền vào trường này"
"Tên đăng nhập hoặc mật khẩu không đúng!"

Do not invent other messages.

==================================================
OUTPUT FORMAT
==================================================

Return JSON only.

Structure:

{
  "items": [
    {
      "Testcase": "LG01",
      "Username": "...",
      "Password": "...",
      "Expected": "..."
    }
  ]
}

Rules:
- Use sequential IDs: LG01, LG02, ...
- Each item must contain exactly these four fields:
  - Testcase
  - Username
  - Password
  - Expected
- Do not output explanations
- Do not output markdown
- Do not output extra keys

==================================================
FINAL CHECK BEFORE OUTPUT
==================================================

Before producing the final JSON, check that the test set includes distinct coverage for:
- successful login
- missing Username
- missing Password
- both fields missing if it creates distinct behavior
- wrong Username
- wrong Password
- both wrong if it creates distinct behavior
- at least some representative edge/error-prone inputs

Remove redundant cases that repeat the same logic path without adding new coverage.
